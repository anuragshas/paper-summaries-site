Can AI Agents Generate Microservices? How Far are We?

Authors: Bassam Adnan, Matteo Esposito, Davide Taibi, Karthik Vaidhyanathan

Source: https://arxiv.org/pdf/2603.09004

Generated summary artifact for 2603.09004

Thesis

AI agents are capable of generating microservices with maintainable code, but their performance in terms of correctness is inconsistent and highly dependent on contextual scenarios and prompt strategies, indicating that fully autonomous microservice generation is not yet achievable without significant human oversight.

Core Problem

While Large Language Models (LLMs) have advanced code generation, their application in generating complex microservices with explicit dependencies and API contracts remains largely unexplored and the influence of contextual information on their performance is understudied.

Key Novelty

The first comprehensive empirical study on AI agents generating microservices, evaluating functional correctness, code quality, and efficiency across different contextual scenarios and prompt strategies.

Key Contributions

  • AI agents successfully generated microservices with functional correctness, achieving up to 98% pass rates in clean state scenarios.
  • Generated code consistently exhibited lower Cyclomatic Complexity and Cognitive Complexity (15-40% reduction) compared to human baselines, suggesting simpler and more maintainable code structures.
  • Prompt strategies had scenario-dependent effects: minimal prompts (P1) generally outperformed detailed summaries (P2) in incremental generation, while P2 sometimes significantly benefited clean state generation (e.g., Code Qwen's 63.7 percentage point improvement in integration success).
  • Significant efficiency disparities were observed: Code Qwen was the most cost-effective ($2.98/service) and Claude Code was the most expensive ($13.28/service). Codex had the longest generation times, with outliers reaching 104 minutes.
  • Performance was degraded on proprietary codebases compared to open-source ones, indicating potential data contamination effects from LLM training data.

Results

  • AI agents can generate functional microservices, but their performance varies significantly by contextual scenario and prompt strategy.
  • In incremental generation, minimal prompts (P1) generally led to better performance by encouraging exploration, while detailed summaries (P2) often degraded performance for Claude Code and Codex.
  • Clean state generation consistently achieved high functional correctness (up to 98%), with explicit guidance (P2) sometimes providing substantial benefits (e.g., for Code Qwen).
  • Generated code generally exhibits lower cyclomatic and cognitive complexity compared to human baselines, suggesting simpler and more maintainable structures.
  • Significant differences in efficiency were observed across agents: Claude Code and Code Qwen were faster, while Codex was slower with notable outliers. Code Qwen was the most cost-effective, and Claude Code the most expensive.
  • Token verbosity did not correlate with functional correctness; Claude Code generated concise code with comparable or better test pass rates.
  • Performance was lower on proprietary codebases compared to open-source ones, indicating potential data contamination effects from LLM training.

Limitations

  • External validity is limited by the selection of specific AI agents and generation parameters, and generalizability to other architectural patterns beyond REST-based microservices is restricted.
  • Data contamination from LLM training on open-source repositories could inflate performance, as agents consistently performed worse on private projects not exposed to public training data.
  • Internal validity issues arise from using different test strategies (unit vs. integration tests) for different scenarios, which validate distinct aspects of correctness, and the pilot study potentially optimizing Claude Code's performance.
  • Construct validity is affected by the selection of specific microservices for removal, and efficiency metrics, though measured under controlled conditions, might be affected by variations in cloud service latency not accounted for.
  • Prompt caching mechanisms were not accounted for, potentially overstating actual costs for agents with effective caching.
